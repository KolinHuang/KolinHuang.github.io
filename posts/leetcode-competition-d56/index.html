5792. 统计平方和三元组的数目

一个 平方和三元组 (a,b,c) 指的是满足 a2 + b2 = c2 的 整数 三元组 a，b 和 c 。

给你一个整数 n ，请你返回满足 1 <= a, b, c <= n 的 平方和三元组 的数目。

就是找符合勾股定理的组合数，a\b\c肯定不相等。

暴力优化：

  1. 两边之和大于第三边
  2. 像3,4,5是一组，那4,3,5也是一组，其实这一组是重复计算的。那么可以让a < b < c，然后找到符合条件的，就+2，因为（a,b,c）和（b,a,c）都符合。

class Solution {
    public int countTriples(int n) {
        int res = 0;
        for(int a = 1; a <= n; ++a){
            for(int b = a+1; b <= n; ++b){
                for(int c = b+1; c <= n; ++c){
                    if(a + b < c)   break;
                    if(a * a + b * b == c * c){
                        res += 2;
                    }
                }
            }
        }
        return res;
    }
}

5793. 迷宫中离入口最近的出口

给你一个 m x n 的迷宫矩阵 maze （下标从 0 开始），矩阵中有空格子（用 ‘.’ 表示）和墙（用 ‘+’ 表示）。同时给你迷宫的入口 entrance ，用 entrance = [entrancerow, entrancecol] 表示你一开始所在格子的行和列。

每一步操作，你可以往 上，下，左 或者 右 移动一个格子。你不能进入墙所在的格子，你也不能离开迷宫。你的目标是找到离 entrance 最近 的出口。出口 的含义是 maze 边界 上的 空格子。entrance 格子 不算 出口。

请你返回从 entrance 到最近出口的最短路径的 步数 ，如果不存在这样的路径，请你返回 -1 。

输入：maze = [[”+”,”+”,”.”,”+”],[”.”,”.”,”.”,”+”],[”+”,”+”,”+”,”.”]], entrance = [1,2] 输出：1 解释：总共有 3 个出口，分别位于 (1,0)，(0,2) 和 (2,3) 。 一开始，你在入口格子 (1,2) 处。

  • 你可以往左移动 2 步到达 (1,0) 。
  • 你可以往上移动 1 步到达 (0,2) 。 从入口处没法到达 (2,3) 。 所以，最近的出口是 (0,2) ，距离为 1 步。

又是BFS，这种题目，我总是想到DFS，还是BFS做太少了。

class Solution {

    public int nearestExit(char[][] maze, int[] entrance) {
        int m = maze.length-1;
        int n = maze[0].length-1;
        boolean[][] visited = new boolean[m+1][n+1];
        visited[entrance[0]][entrance[1]] = true;
        Queue<int[]> queue = new LinkedList<>();
        queue.offer(entrance);
        int step = 0;
        int[][] direction = [[1,0],[-1,0],[0,1],[0,-1]];
        while(!queue.isEmpty()){
            int size = queue.size();
            step++;
            for(int i = 0; i < size; ++i){
                int[] tmp = queue.poll();
                int x = tmp[0];
                int y = tmp[1];
                
                if(x == 0 || x == m || y == 0 || y == n){
                    if(!(x == entrance[0] && y == entrance[1])){
                        return step-1;
                    }
                }
                for(int j = 0; j < 4; ++j){
                    int new_x = x + direction[j][0];
                    int new_y = y + direction[j][1];
                    if(new_x < 0 || new_x > m || new_y < 0 || new_y > n)    continue;
                    if(maze[new_x][new_y] == '+')   continue;
                    if(visited[new_x][new_y])   continue;
                    visited[new_x][new_y] = true;
                    queue.offer(new int[]{new_x, new_y});
                }
            }
        }
        return -1;
    }
    
}

5794. 求和游戏

Alice 和 Bob 玩一个游戏，两人轮流行动，Alice 先手 。

给你一个 偶数长度 的字符串 num ，每一个字符为数字字符或者 ‘?’ 。每一次操作中，如果 num 中至少有一个 ‘?’ ，那么玩家可以执行以下操作：

选择一个下标 i 满足 num[i] == ‘?’ 。 将 num[i] 用 ‘0’ 到 ‘9’ 之间的一个数字字符替代。 当 num 中没有 ‘?’ 时，游戏结束。

Bob 获胜的条件是 num 中前一半数字的和 等于 后一半数字的和。Alice 获胜的条件是前一半的和与后一半的和 不相等 。

比方说，游戏结束时 num = “243801” ，那么 Bob 获胜，因为 2+4+3 = 8+0+1 。如果游戏结束时 num = “243803” ，那么 Alice 获胜，因为 2+4+3 != 8+0+3 。 在 Alice 和 Bob 都采取 最优 策略的前提下，如果 Alice 获胜，请返回 true ，如果 Bob 获胜，请返回 false 。

输入：num = "5023"
输出：false
解释：num 中没有 '?' ，没法进行任何操作。
前一半的和等于后一半的和：5 + 0 = 2 + 3 。

我尝试解释一下为什么“sum是9的倍数”，Bob才有机会赢。如果sum不是9的倍数，站在Alice的角度看，Bob选择了1~9内最优的数字k来补齐差值，Alice只需要选择9-k来将Bob这次的操作抵消掉就可以了。但是Alice是先选的一方，那么她只要一直选9/-9，Bob就没办法补齐差值。条件“sum == cnt/2 * -9”保证了Bob是有能力补齐差值的。

class Solution {
    public boolean sumGame(String num) {
        //先求和，结果分别是a和b，然后说前一半有x个格子，后一半有y个格子
        //A和B轮流填这x和y个格子，只要Alice能获胜，就返回TRUE
        /**
         * 假设left == right, 那么左右差值为0，剩余格子数只要两边不相等，那么Alice就赢了，格子数相等，Bob就赢了
         * 假设left < right, 那么左右差值为负数，左右抵消后的剩余格子数为k，
         *                                  假设k在左边，那么Bob需要全力去补漏洞，Alice无能为力，只能看着
         *                                      只要k/2 * 9 >= right - left，Bob就能赢
         *                                  假设k在右边，那么Bob无论怎么补，都补不上了
         * 假设left > right, 那么左右差值为正数，左右抵消后的剩余格子数为k,
         *                                  假设k在左边，Bob补不上，
         *                                  假设k在右边，只要k/2 * 9 >= left - right，Bob就能赢
         *
         */
        int n = num.length(), sum = 0, cnt = 0;
        for(int i = 0;i < n>>1; i++){
            char c = num.charAt(i);
            if(c == '?'){
                cnt++;
            }else{
                sum += c^48;
            }
        }
        for(int i=n>>1;i<n;i++){
            char c = num.charAt(i);
            if(c == '?'){
                cnt--;
            }else{
                sum -= c^48;
            }
        }
        if((cnt)%2 == 0 && sum == cnt/2 * -9) return false;
        return true;
    }
}

5795. 规定时间内到达终点的最小花费

一个国家有 n 个城市，城市编号为 0 到 n - 1 ，题目保证 所有城市 都由双向道路 连接在一起 。道路由二维整数数组 edges 表示，其中 edges[i] = [xi, yi, timei] 表示城市 xi 和 yi 之间有一条双向道路，耗费时间为 timei 分钟。两个城市之间可能会有多条耗费时间不同的道路，但是不会有道路两头连接着同一座城市。

每次经过一个城市时，你需要付通行费。通行费用一个长度为 n 且下标从 0 开始的整数数组 passingFees 表示，其中 passingFees[j] 是你经过城市 j 需要支付的费用。

一开始，你在城市 0 ，你想要在 maxTime 分钟以内 （包含 maxTime 分钟）到达城市 n - 1 。旅行的 费用 为你经过的所有城市 通行费之和 （包括 起点和终点城市的通行费）。

给你 maxTime，edges 和 passingFees ，请你返回完成旅行的 最小费用 ，如果无法在 maxTime 分钟以内完成旅行，请你返回 -1 。

这题我第一思路是迪杰斯特拉，但是有maxTime的限制。所以就不会了~

其实就是在最短路的基础上加了一维时间的限制而已，所以对于dis[i]我们对它加一维来限制就好了。
dis[i][cost]表示到达i点时间花费cost的最小费用。
然后就是dijkstra的模板进行修改就好了。
我们根本不关心时间花费了多少，只要在限制内就好，需要在把点加入队列的时候进行判断时间是否满足即可。
这其实是叫做分层图最短路的东西，有兴趣可以自行百度了解。
