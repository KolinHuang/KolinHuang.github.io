并发：如何优雅地关闭线程

Posted Jul 13 2021-07-13T18:05:00+08:00 by Kol Huang

事发得很突然，让我手写一个令牌桶限流算法。期间把我一顿吊锤。事后说，这样的实现会造成内存泄漏，请实现一种线程优雅的退出方式~卒

为了下次不再栽在这里，于是整理记录了一下关闭线程的方式。

方式一：设置线程中断标志

两阶段终止：

  1. 准备阶段：发出终止指令，通过设置中断标志,并发送中断信号，“通知”目标线程，可以准备停止了。
  2. 执行阶段：响应终止指令，接收到中断信号及标志，在此基础决定线程退出时机，并执行适当清理工作。

当前线程被 sleep、Thread.wait、Thread.join 三种方法之一阻塞时，调用当前线程的 interrupt 方法会抛出 InterruptedException 异常。

在准备阶段，我们需要做两件事情:

设置中断标志：中断标志的作用是标识线程已经中断了，当线程读到这个标识后，就可以执行退出操作了。

发出中断信号：光设置中断标志还不行，如果线程当前处于阻塞状态，就算设置了中断标志，线程也无法检测到，这时就需要我们调用目标线程的中断方法interrupt()，将其从阻塞状态中唤醒，而目标线程通过捕获InterruptedException异常，来侦测这个中断信号。

/**
 * @author kol Huang
 * @date 2021/7/13
 */
public class MyThread extends Thread {

    @Override
    public void run() {
        super.run();

            //TODO do Job.
            try {
                while (true) {
                    Thread.sleep(1000);
                    System.out.println("线程正在执行中...");
                }
            } catch (InterruptedException e) {
                e.printStackTrace();
                System.out.println("线程退出...");
            }finally {
                System.out.println("善后工作处理完毕。");
            }
    }
    public void close(){
        this.interrupt();
        System.out.println("开始关闭线程..");
    }
}

上述方法依赖于线程的中断标志，如果其他线程调用了interrupted清除了中断标志，那么线程会中断失败。

同时有些阻塞是不会抛异常的，例如LockSupport的park方法，也不适用。所以可以自定义中断标志位。

方式二：自定义中断标志

/**
 * @author kol Huang
 * @date 2021/7/13
 */
public class MyThread extends Thread {

    private volatile boolean isTerminated = false;

    @Override
    public void run() {
        super.run();
        while (!isTerminated) {
            //TODO do Job.
            try {
                Thread.sleep(1000);
                System.out.println("线程正在执行中...");
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        System.out.println("线程退出...");
    }

    public void close(){
        this.isTerminated = true;
        System.out.println("开始关闭线程..");
    }

}
